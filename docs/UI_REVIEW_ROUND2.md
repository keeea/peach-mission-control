# UI Review Round 2

聚焦页面：Dashboard / Kanban / Reports

## 1. 过滤器上下文不够显性
- **问题**：当前仅显示表单控件，缺少“当前筛选范围”的持续提示；无结果时 project 选项还会变少。
- **影响**：用户会忘记自己为何“看不到内容”，并误判为空数据而不是被筛选掉。
- **改进**：增加 active filter chips、scope 文案、clear single / clear all；project 下拉始终保留完整可选项。
- **验收**：任意页面应用筛选后，可立即看见当前上下文；单个条件可移除；无结果时筛选器仍完整可用。

## 2. 空状态缺少行动导向
- **问题**：空列表只给一句 muted 文案，没有下一步建议。
- **影响**：页面在“没有任务 / 没有筛选结果”时像是坏掉，而不是可继续操作。
- **改进**：为 dashboard、kanban、reports 增加 empty state，明确区分“无数据”与“无匹配结果”，并提供 reset / create / open board 等 CTA。
- **验收**：三页在空状态下都能给出原因、建议动作和直达入口。

## 3. 卡片信息权重还不够成熟
- **问题**：任务卡和列表项中标题、元信息、操作按钮层级接近，阅读焦点分散。
- **影响**：扫描速度慢，用户很难一眼分辨“主任务标题 / 次级属性 / 可操作项”。
- **改进**：压缩次级信息视觉噪音，强化标题、状态、风险标签；统一间距、对齐和卡片密度。
- **验收**：任务标题成为首视觉锚点，meta 信息退后一级，操作按钮只在 hover / focus 或结构化位置出现。

## 4. Hover / focus / active 反馈不一致
- **问题**：不同按钮、链接、卡片、dropzone 的交互反馈强弱不一致，focus 可见性偏弱。
- **影响**：产品质感不足，键盘使用与精细操作体验不稳定。
- **改进**：统一 hover elevation、focus ring、pressed/active 语言；为导航、按钮、chips、任务卡建立一致状态反馈。
- **验收**：主要可交互组件均具备可见 hover/focus 状态，视觉节奏一致。

## 5. Kanban 缺少真正“快操作”
- **问题**：虽然支持拖拽和详情编辑，但缺少轻量级 status quick action，常见小改动路径偏长。
- **影响**：用户为了做微调，必须 drag 或打开详情，降低日常更新效率。
- **改进**：在卡片内加入快速状态切换入口与 inline save 状态反馈，保留详情弹窗处理重编辑。
- **验收**：用户可在卡片层 1~2 次点击完成常见状态流转，并得到即时反馈。

## 6. Reports 表达过于“原始数据”
- **问题**：breakdown 直接输出 dict/code，可读性弱，像 debug panel。
- **影响**：报告不像可读汇报，更像接口回包，不利于快速吸收信息。
- **改进**：改为 summary blocks / ranked list / readable labels，并补充空状态说明。
- **验收**：报告页可在 10 秒内看懂本周完成、阻塞、负责人分布和项目分布。
