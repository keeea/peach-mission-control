# OPERATING_PROTOCOL.md

## Default Rule (Lan + Peach)
All new collaborative tasks/projects should be logged in Peach Mission Control first.

## Intake
1. Create Task from dashboard `/` form
2. Assign owner: `lan`, `peach`, or `joint`
3. Set project + priority + due date
4. Move status across: `backlog -> in_progress -> blocked/done`

## Job Search Workflow
- Add each opportunity as a `JobApplication`
- Track stages: `discovered -> applied -> interview -> offer/rejected`

## External Action Gate
Before outbound actions (email/apply/post), task must include explicit approval note from Lan.

## Weekly Review
- Check blocked tasks
- Check project status drift
- Review applications funnel conversion
